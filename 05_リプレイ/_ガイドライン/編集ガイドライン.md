# 編集ガイドライン - リプレイ小説の読み物クオリティ向上

**作成日:** 2026-01-01
**対象:** 編集者エージェント（Editor Agent）
**目的:** リプレイ小説を「読み物」として面白く、読者を引き込む作品に仕上げる

---

## 📖 編集者の役割

編集者エージェントは、原作者が執筆したリプレイ小説を**読者視点**でレビューし、以下の観点から改善提案を行います：

1. **Show vs Tellのバランス** - 説明ではなく描写で伝える
2. **テンポ・リズム** - 緩急をつけて読者を飽きさせない
3. **冗長性の排除** - 無駄な繰り返しを削減
4. **段落構成** - 読みやすい長さと改行
5. **クリフハンガーの効果** - 次回への引きを強化
6. **感情誘導** - 読者の感情を適切に動かす

編集者は**原作者と協働**し、事実関係を尊重しながらも、読み物としての完成度を高めます。

---

## 1. Show vs Tell のバランス

### 基本原則

**Tell（説明）:** 情報を直接伝える
**Show（描写）:** 行動や描写で間接的に伝える

**重要:** リプレイ小説は事実に基づくため、完全にShowのみにはできません。Tellも必要ですが、バランスが重要です。

### ❌ NG例（Tellが多すぎる）

```markdown
センサ船長は困っているようだった。彼は墓荒らしの件を説明した。シャルルは正義感が強いので、すぐに行動することにした。
```

**問題点:**
- 「困っているようだった」→ 感情を直接説明
- 「説明した」→ 会話の内容を要約
- 「正義感が強いので」→ キャラクターの性格を説明

### ✅ OK例（Showを増やす）

```markdown
センサ船長の眉間には深い皺が刻まれていた。「墓が荒らされているんだ。死者への冒涜だ」その声には、怒りと悲しみが滲んでいた。

「見過ごすわけにはいかない！」シャルルの声には、揺るぎない決意が宿っていた。彼の手は剣の柄を強く握りしめている。
```

**改善点:**
- 「眉間の皺」「声の調子」で感情を描写
- 直接会話で内容を伝える
- 「剣の柄を握る」行動でシャルルの決意を示す

### Tell を使うべき場合

以下の場合はTellも有効です：

1. **時間経過を要約する**
   - ✅ 「一行は墓地へと向かった」（移動シーンの要約）
   - ❌ 「一行は歩いて、途中で休憩して、また歩いて...」（冗長）

2. **複雑な設定を説明する**
   - ✅ 「ファルジーン市は、かつて栄えた港町だった」（世界観の説明）

3. **テンポを重視する**
   - ✅ 「戦闘は一瞬で決着した」（重要でない戦闘の要約）

### 編集レビュー時のチェックポイント

**チェック項目:**
- [ ] 感情を直接説明していないか（「嬉しかった」「怒った」など）
- [ ] 会話を要約していないか（「説明した」「話し合った」など）
- [ ] キャラクターの性格を説明していないか（「正義感が強い」など）
- [ ] 重要なシーンでShowが使われているか
- [ ] Tell を使う場合、理由が明確か（時間経過、設定説明など）

**フィードバック例:**
```markdown
### 指摘: Show vs Tell

**箇所:**
> センサ船長は困っているようだった。

**問題:** Tellが多く、臨場感に欠ける

**修正案:**
> センサ船長の眉間には深い皺が刻まれていた。その声には、怒りと悲しみが滲んでいた。

**理由:** 感情を描写で示すことで、読者がセンサ船長の心情をより深く感じられる
```

## 2. テンポ・リズム

### 基本原則

リプレイ小説には**緩急**が必要です。すべてのシーンを同じ密度で描写すると、読者が飽きてしまいます。

**重要シーン:** 詳細に描写（戦闘、重要な会話、伏線の提示）
**移動シーン:** 簡潔に要約（テンポを重視）

### ❌ NG例（テンポが悪い）

```markdown
一行は休憩所を出た。街の通りを歩いた。市場を通り過ぎた。橋を渡った。丘を登った。そして墓地に到着した。

墓地に着くと、巡礼者は周囲を見渡した。ノクシルも辺りを確認した。シャルルも同じように周りを見た。
```

**問題点:**
- 移動シーンが冗長（6文で移動を描写）
- 同じ行動の繰り返し（全員が周囲を見る）

### ✅ OK例（テンポを改善）

```markdown
一行は休憩所を発ち、街を抜けて墓地へと向かった。

丘の上に広がる墓地。巡礼者の鋭い視線が、荒らされた墓石の列を捉えた。「この土は最近掘り返されている」ノクシルが低い声で囁く。シャルルは剣の柄に手をかけた。
```

**改善点:**
- 移動を1文で要約
- 墓地到着後のシーンを詳細に描写（重要シーン）
- 各キャラクターの行動に個性を持たせる

### シーンの重要度判定

| シーン種別 | 重要度 | 描写の濃さ | 文字数目安 |
|---|---|---|---|
| **戦闘** | 高 | 濃い | 300-800字 |
| **重要な会話** | 高 | 濃い | 200-500字 |
| **伏線の提示** | 高 | 濃い | 100-300字 |
| **調査・探索** | 中 | 中程度 | 100-300字 |
| **移動** | 低 | 薄い | 30-100字 |
| **日常会話** | 低 | 薄い | 50-150字 |

### 編集レビュー時のチェックポイント

**チェック項目:**
- [ ] 移動シーンが長すぎないか（100字以上の場合は要確認）
- [ ] 重要なシーン（戦闘、重要な会話）が詳細に描写されているか
- [ ] 同じペースが続きすぎていないか（緩急のメリハリ）
- [ ] 各章の長さがバランス良いか（極端に短い/長い章がないか）

**フィードバック例:**
```markdown
### 指摘: テンポ

**箇所:** 墓地への移動シーンが2段落（約400字）

**問題:** 重要でない移動シーンが長すぎる

**修正案:** 1段落（約100字）に圧縮、または完全にカット

**理由:** 移動シーンは物語の進行に寄与しないため、簡潔にしてテンポを維持すべき
```

## 3. 冗長性の排除

### 基本原則

**冗長性:** 同じ情報を繰り返す、不要な修飾語、無駄な説明

読者は同じ情報を何度も読みたくありません。簡潔に伝えることが重要です。

### ❌ NG例（冗長）

```markdown
シャルルは正義感が強い。彼は墓荒らしを許せないと思った。そのため、すぐに行動することにした。彼は仲間たちに声をかけた。「行こう」とシャルルは言った。
```

**問題点:**
- 「正義感が強い」→「許せない」→「行動」（同じ内容を3文で）
- 「声をかけた」→「言った」（重複）

### ✅ OK例（簡潔）

```markdown
「見過ごすわけにはいかない！」シャルルの声には、揺るぎない決意が宿っていた。
```

**改善点:**
- セリフと描写で1文に集約
- 正義感、決意、行動がすべて伝わる

### よくある冗長パターン

#### パターン1: 感情の重複説明

❌ 「巡礼者は驚いた。彼は予想外の出来事に目を見開いた」
✅ 「巡礼者は目を見開いた」（描写で十分）

#### パターン2: 同じ語句の繰り返し

❌ 「墓地は静かだった。とても静かで、音一つしなかった」
✅ 「墓地は静まり返っていた」

#### パターン3: 不要な修飾語

❌ 「彼は素早く急いで走った」（「素早く」と「急いで」が重複）
✅ 「彼は駆け出した」

#### パターン4: セリフと地の文の重複

❌ 「『行こう』とシャルルは言った。彼は仲間たちに行くことを提案した」
✅ 「『行こう』とシャルルは言った」（セリフで十分）

### 編集レビュー時のチェックポイント

**チェック項目:**
- [ ] 同じ情報を2回以上伝えていないか
- [ ] 不要な修飾語がないか（「とても」「非常に」など）
- [ ] セリフと地の文で同じ内容を繰り返していないか
- [ ] 3文を1文にできないか

**フィードバック例:**
```markdown
### 指摘: 冗長性

**箇所:**
> シャルルは正義感が強い。彼は墓荒らしを許せないと思った。そのため、すぐに行動することにした。

**問題:** 同じ内容を3文で繰り返している

**修正案:**
> 「見過ごすわけにはいかない！」シャルルの声には、揺るぎない決意が宿っていた。

**理由:** セリフと描写で1文に集約することで、テンポが向上し、読者に強い印象を与えられる
```

## 4. 段落構成

### 基本原則

**段落の役割:** 読者に視覚的な「休憩」を提供し、内容をまとまりで区切る

**適切な段落:**
- 1段落 = 1つのアイデア・シーン
- 長さ: 50-200字程度（2-5文）
- 長すぎる段落は分割、短すぎる段落は統合

### ❌ NG例（段落が長すぎる）

```markdown
巡礼者は墓地に到着した。周囲を見渡すと、荒らされた墓石が目に入った。この土は最近掘り返されている。ノクシルも辺りを確認している。彼は「問題ない、追跡できる」と言った。シャルルは剣の柄に手をかけた。彼は警戒している様子だ。墓地は静かで、不穏な空気が漂っている。突然、奥から物音がした。巡礼者は身構えた。（約150字、10文）
```

**問題点:**
- 1段落に10文が詰め込まれている
- 複数のシーン（到着、調査、警戒、物音）が混在
- 視覚的に読みにくい

### ✅ OK例（段落を分割）

```markdown
巡礼者は墓地に到着した。周囲を見渡すと、荒らされた墓石が目に入った。「この土は最近掘り返されている」

「問題ない、追跡できる」ノクシルが低い声で囁く。シャルルは剣の柄に手をかけた。墓地は静まり返っている。

突然、奥から物音がした。巡礼者は身構えた。
```

**改善点:**
- 3段落に分割（到着・調査 / 警戒 / 物音）
- 各段落が1つのシーンに集中
- 視覚的に読みやすい

### 段落分割のタイミング

以下のタイミングで段落を分けます：

1. **視点が変わる**
   - 巡礼者の行動 → ノクシルの行動

2. **シーンが変わる**
   - 調査シーン → 戦闘シーン

3. **時間が経過する**
   - 「数分後」「やがて」など

4. **話者が変わる**
   - キャラクターAのセリフ → キャラクターBのセリフ

5. **感情が変化する**
   - 緊張 → 安堵

### 改行のルール

**セリフの前後は改行:**
```markdown
✅ OK:
巡礼者は墓地に到着した。

「この土は最近掘り返されている」

ノクシルが頷く。

❌ NG:
巡礼者は墓地に到着した。「この土は最近掘り返されている」ノクシルが頷く。
```

**長いセリフは独立させる:**
```markdown
✅ OK:
巡礼者は深く息を吐いた。

「我々は長い旅を続けてきた。多くの困難に直面し、仲間を失い、それでも進んできた。だが、この先に何が待ち受けているのか、誰にも分からない」

シャルルは頷いた。
```

### 編集レビュー時のチェックポイント

**チェック項目:**
- [ ] 1段落が200字を超えていないか
- [ ] 1段落に1つのアイデア・シーンが集中しているか
- [ ] セリフの前後で改行されているか
- [ ] 視点・シーン・時間の変化で段落が分かれているか
- [ ] 極端に短い段落（1文のみ）が連続していないか

**フィードバック例:**
```markdown
### 指摘: 段落構成

**箇所:** 第2章の調査シーン（約250字、1段落）

**問題:** 1段落が長すぎて、複数のシーンが混在している

**修正案:** 以下の3段落に分割
1. 墓地到着と初期調査
2. ノクシルの追跡とシャルルの警戒
3. 物音の発見

**理由:** 各シーンを独立させることで、読者が内容を理解しやすくなり、テンポも向上する
```

## 5. クリフハンガーの効果

### 基本原則

**クリフハンガー:** 次回への引き、読者に「続きが気になる」と思わせる技法

リプレイ小説の最後には、効果的なクリフハンガーを配置して、次回への期待感を高めます。

### クリフハンガーの種類

#### タイプ1: 謎の提示

```markdown
✅ 例:
地下通路の奥に、不気味な光が揺らめいている。巡礼者は一歩踏み出した。そこに何が待ち受けているのか、まだ誰も知らない。
```

**効果:** 読者が「光の正体は何？」と気になる

#### タイプ2: 危機の到来

```markdown
✅ 例:
突然、背後から複数の足音が響いた。振り返ると、闇の中から無数の影が迫ってくる。ノクシルは弓を構えた。「囲まれた」
```

**効果:** 読者が「どうなるの？」と緊張する

#### タイプ3: 重要な決断

```markdown
✅ 例:
「我々はここで引き返すべきか、それとも先に進むべきか」シャルルの問いに、巡礼者は黙って地下通路を見つめた。その奥には、彼らの運命を変える何かが待っているはずだ。
```

**効果:** 読者が「どう決断するの？」と期待する

#### タイプ4: 衝撃の事実

```markdown
✅ 例:
「待て」ノクシルが声を上げた。「この紋章は...まさか、ゴート教団のものか？」一同は息を呑んだ。
```

**効果:** 読者が「ゴート教団が関係している！？」と驚く

### ❌ NG例（弱いクリフハンガー）

```markdown
一行は地下通路を発見した。次回は地下通路を探索する予定だ。
```

**問題点:**
- メタ的な表現（「次回は～予定だ」）
- 期待感がない
- 淡々としている

### ✅ OK例（強いクリフハンガー）

```markdown
地下通路の奥から、微かに聞こえる詠唱の声。それは人間のものではない。巡礼者は剣を抜き、闇の中へと踏み出した。
```

**改善点:**
- 謎の提示（詠唱の声の正体）
- 緊張感（人間ではない）
- 行動の開始（踏み出した）

### クリフハンガーの配置

**章の最後:**
各章の最後にも小さなクリフハンガーを配置すると、読者が次の章に進みやすくなります。

```markdown
✅ 例（第1章の最後）:
墓地への道は遠い。だが、巡礼者の目には、既に覚悟が宿っていた。この旅の先に、何が待ち受けているのか。
```

**リプレイ全体の最後:**
次回への強い引きを配置します。

```markdown
✅ 例（第X回の最後）:
地下通路の奥に、不気味な光が揺らめいている。その光は、まるで彼らを誘うかのように、ゆっくりと明滅していた。巡礼者は一歩踏み出した。

そこに何が待ち受けているのか、まだ誰も知らない。
```

### 編集レビュー時のチェックポイント

**チェック項目:**
- [ ] リプレイの最後にクリフハンガーが配置されているか
- [ ] クリフハンガーの種類は適切か（謎、危機、決断、事実）
- [ ] 「次回は～」などメタ的な表現を使っていないか
- [ ] 読者が「続きが気になる」と思えるか
- [ ] 各章の最後にも小さなクリフハンガーがあるか

**フィードバック例:**
```markdown
### 指摘: クリフハンガー

**箇所:** リプレイの最後（第4章）

**問題:** クリフハンガーが弱く、次回への期待感が薄い

**現状:**
> 一行は地下通路を発見した。次回は地下通路を探索する予定だ。

**修正案:**
> 地下通路の奥から、微かに聞こえる詠唱の声。それは人間のものではない。巡礼者は剣を抜き、闇の中へと踏み出した。

**理由:** 謎の提示（詠唱の声）と緊張感（人間ではない）で、読者が次回を期待できる
```

## 6. 感情誘導

### 基本原則

**感情誘導:** 読者の感情を意図的に動かす技法

リプレイ小説では、読者に以下の感情を抱かせることを目指します：

- **緊張感:** 戦闘シーン、危機的状況
- **安堵:** 危機脱出、仲間との再会
- **驚き:** 予想外の展開、衝撃の事実
- **共感:** キャラクターの葛藤、内面描写
- **期待感:** 次回への引き

### 緊張感の高め方

#### テクニック1: 短文を連続させる

```markdown
✅ 例:
物音がした。巡礼者は振り返った。何かが動いている。影が迫る。ノクシルが矢を番えた。「来る」
```

**効果:** 短文のリズムで緊張感を演出

#### テクニック2: 五感を使う

```markdown
✅ 例:
冷たい風が頬を撫でた。土の湿った匂い。足元の石が小さく音を立てる。そして、闇の奥から聞こえる、何かが這いずる音。
```

**効果:** 五感描写で読者をシーンに引き込む

#### テクニック3: 時間を引き延ばす

```markdown
✅ 例:
巡礼者は剣を構えた。敵の動きを見極める。一瞬の沈黙。そして、敵が飛びかかった。
```

**効果:** 時間を引き延ばして緊張を高める

### 安堵の演出

#### テクニック1: 長文に戻す

```markdown
✅ 例:
戦闘が終わった。巡礼者は深く息を吐き、剣を鞘に収めた。シャルルは座り込み、額の汗を拭う。ノクシルは静かに周囲を警戒しているが、その肩の力は抜けている。
```

**効果:** 長文のリズムで安堵感を演出

#### テクニック2: 温かい描写

```markdown
✅ 例:
「無事でよかった」シャルルの声には、安堵が滲んでいた。巡礼者は頷く。仲間の無事を確認し、彼の心にも静かな安らぎが広がる。
```

**効果:** 温かい言葉と描写で安堵を伝える

### 驚きの演出

#### テクニック1: 意外な事実を最後に配置

```markdown
✅ 例:
「この紋章は...」ノクシルの声が震えた。「まさか、ゴート教団のものか？」

一同は息を呑んだ。彼らが追っていた墓荒らしは、単なる盗賊ではなかったのだ。
```

**効果:** 重要な事実を段階的に明かして驚きを演出

### 共感の演出

#### テクニック1: 内面描写を丁寧に

```markdown
✅ 例:
シャルルは拳を握りしめた。正義を貫くと誓った。だが、その正義が本当に正しいのか、彼自身にも分からない。ルンジャタの顔が脳裏に浮かぶ。「俺は、本当に...」
```

**効果:** キャラクターの葛藤を描くことで、読者の共感を得る

**注意:** 内面描写はプレイヤーログに基づくこと

### 編集レビュー時のチェックポイント

**チェック項目:**
- [ ] 戦闘シーンで緊張感が高まっているか
- [ ] 危機脱出後に安堵が描写されているか
- [ ] 重要な事実の提示で驚きを演出できているか
- [ ] キャラクターの内面描写で共感を得られているか
- [ ] 感情の起伏（緩急）があるか

**フィードバック例:**
```markdown
### 指摘: 感情誘導

**箇所:** 第3章の戦闘シーン

**問題:** 緊張感が薄い。長文が続いて、戦闘の臨場感に欠ける

**修正案:** 短文を連続させて緊張感を高める

**修正前:**
> 巡礼者は剣を構えて、敵に向かって走り出し、そして敵を斬りつけた。

**修正後:**
> 巡礼者は剣を構えた。敵が飛びかかる。一閃。剣が敵を捉えた。

**理由:** 短文のリズムで戦闘の緊迫感を演出し、読者を引き込む
```

## 🎯 編集者としての心構え

1. **原作者を尊重する**
   - 批判ではなく、建設的な提案を
   - 「ダメ」ではなく「こうするとより良い」

2. **事実関係は譲らない**
   - 編集案が事実と異なる場合は、一次データを優先
   - その場合は代替案を提示

3. **読者視点を忘れない**
   - 「自分が読者だったら、これは面白いか？」
   - 「この描写で、読者は感情移入できるか？」

4. **協働を意識する**
   - 編集者と原作者は対立ではなく、協力関係
   - 最終的な目標は「読者に楽しんでもらえる作品」
