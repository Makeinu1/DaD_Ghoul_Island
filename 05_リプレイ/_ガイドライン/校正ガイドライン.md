# 校正ガイドライン - リプレイ小説の正確性担保

**作成日:** 2026-01-01
**対象:** 校正者エージェント（Proofreader Agent）
**目的:** リプレイ小説の文章を正確にし、誤字脱字や表記ゆれを排除する

---

## 📖 校正者の役割

校正者エージェントは、編集済みのリプレイ小説を**文章の正確性**の観点でレビューし、以下のエラーを発見・修正します：

1. **文法エラー** - 助詞の誤用、述語の不一致など
2. **句読点** - 読点・句点の適切な配置
3. **表現のゆらぎ** - 同一語句の異なる表記
4. **漢字・ひらがなバランス** - 読みやすさの確保
5. **誤字脱字** - タイポの発見
6. **同音異義語の誤用** - 「以外/意外」「作る/造る」など

校正者は**ルールベース**で機械的にチェックを行い、原作者に修正案を提示します。

---

## 1. 文法エラー

### 1-1. 助詞の誤用

#### エラー1: 「を」と「に」の誤用

❌ **NG:**
```markdown
彼は墓地を向かった
```

✅ **OK:**
```markdown
彼は墓地に向かった
```

**ルール:** 移動先には「に」を使う

#### エラー2: 「は」と「が」の誤用

❌ **NG:**
```markdown
巡礼者が剣を持っている。巡礼者が敵に向かった。
```

✅ **OK:**
```markdown
巡礼者は剣を持っている。巡礼者が敵に向かった。
```

**ルール:**
- 新情報・強調: 「が」
- 既知情報・主題: 「は」

#### エラー3: 読点で文を繋ぐ（読点の誤用）

❌ **NG:**
```markdown
巡礼者は墓地へ向かった、そしてシャルルも続いた
```

✅ **OK:**
```markdown
巡礼者は墓地へ向かった。シャルルもそれに続いた
```

**ルール:** 読点では文を繋がない。句点で区切る

### 1-2. 述語の不一致

#### エラー1: 「～は...だ」の不一致

❌ **NG:**
```markdown
彼らが見たのは、荒らされた墓だ
```

✅ **OK:**
```markdown
彼らが見たのは、荒らされた墓だった
```

**ルール:** 「見たのは」（過去）に対して「だった」（過去）を使う

#### エラー2: 主語と述語の不一致

❌ **NG:**
```markdown
巡礼者とシャルルは、墓地へ向かう
```

✅ **OK:**
```markdown
巡礼者とシャルルは、墓地へ向かった
```

**ルール:** リプレイ小説は過去の出来事なので、基本的に過去形を使う

### 1-3. 修飾語の位置

#### エラー1: 修飾語の位置が不適切

❌ **NG:**
```markdown
巡礼者は荒らされた墓地に到着した
```

✅ **OK:**
```markdown
巡礼者は墓地に到着した。そこには、荒らされた墓石が並んでいた
```

**ルール:** 「荒らされた」は「墓地」ではなく「墓石」を修飾すべき

### 校正レビュー時のチェックポイント

**文法エラーのチェックリスト:**
- [ ] 助詞（「を」「に」「は」「が」）が正しく使われているか
- [ ] 読点で文を繋いでいないか
- [ ] 述語が主語・時制と一致しているか
- [ ] 修飾語の位置が適切か

**校正レビュー例:**
```markdown
## 文法エラー

### エラー1
**箇所:** 「巡礼者は墓地へ向かった、そしてシャルルも続いた」
**問題:** 読点で文を繋いでいる（読点の誤用）
**修正案:** 「巡礼者は墓地へ向かった。シャルルもそれに続いた」
```

## 2. 句読点

### 2-1. 読点（、）の適切な配置

#### ルール1: 文が長い場合は読点で区切る

❌ **NG:**
```markdown
巡礼者は剣を構え敵に向かった
```

✅ **OK:**
```markdown
巡礼者は剣を構え、敵に向かった
```

**基準:** 10字以上の文には読点を入れる

#### ルール2: 接続詞の後に読点

❌ **NG:**
```markdown
しかし彼は諦めなかった
```

✅ **OK:**
```markdown
しかし、彼は諦めなかった
```

**対象:** しかし、だが、そして、また、なお、ただし など

#### ルール3: 誤解を避けるための読点

❌ **NG（誤解を招く）:**
```markdown
ここでは戦わない巡礼者が静かに立っている
```

✅ **OK:**
```markdown
ここでは戦わない。巡礼者が静かに立っている
```

または

```markdown
ここでは、戦わない巡礼者が静かに立っている
```

**理由:** 読点なしだと「戦わない巡礼者」と誤解される可能性

#### ルール4: 読点が多すぎる場合は削除

❌ **NG:**
```markdown
巡礼者は、剣を、構え、敵に、向かった
```

✅ **OK:**
```markdown
巡礼者は剣を構え、敵に向かった
```

**基準:** 1文に読点は2-3個まで。それ以上は文を分割する

### 2-2. 句点（。）の適切な配置

#### ルール1: セリフの後の句点

❌ **NG:**
```markdown
「行こう」。と巡礼者は言った
```

✅ **OK:**
```markdown
「行こう」と巡礼者は言った。
```

**理由:** 「と巡礼者は言った」まで含めて1文なので、最後に句点

#### ルール2: セリフで文が終わる場合

✅ **OK:**
```markdown
巡礼者は静かに言った。「行こう」
```

**理由:** セリフで文が終わる場合、セリフ内の句点は不要

### 校正レビュー時のチェックポイント

**句読点のチェックリスト:**
- [ ] 長い文（10字以上）に読点があるか
- [ ] 接続詞の後に読点があるか
- [ ] 読点が多すぎないか（1文に4個以上）
- [ ] セリフの句点が適切か
- [ ] 誤解を招く箇所がないか

**校正レビュー例:**
```markdown
## 句読点

### 問題1
**箇所:** 「巡礼者は剣を構え敵に向かった」
**問題:** 読点がなく読みにくい
**修正案:** 「巡礼者は剣を構え、敵に向かった」
```

## 3. 表現のゆらぎ

### 3-1. 固有名詞の統一

#### ルール1: 人名の統一

❌ **NG:**
```markdown
第1章: センサ船長
第3章: センサー船長
```

✅ **OK:**
```markdown
すべて「センサ船長」に統一
```

**チェック対象:**
- センサ船長 / センサー船長
- ノクシル / ノクシール
- シャルル / シャール
- ディードリド / ディードリッド

#### ルール2: 地名の統一

❌ **NG:**
```markdown
第1章: ファルジーン市
第2章: ファルジーン街
```

✅ **OK:**
```markdown
すべて「ファルジーン市」に統一
```

**チェック対象:**
- ファルジーン市 / ファルジーン街 / ファルジーン島市
- レサンテ港 / レサンテの港

#### ルール3: 組織名の統一

❌ **NG:**
```markdown
第1章: ゴート教団
第3章: ゴートの教団
```

✅ **OK:**
```markdown
すべて「ゴート教団」に統一
```

### 3-2. 一般名詞の統一

#### ルール1: 呪文名の統一

❌ **NG:**
```markdown
第1章: 《シャター》
第2章: 《シャッター》
```

✅ **OK:**
```markdown
すべて「《シャター》」に統一（D&D公式準拠）
```

**チェック対象:**
- シャター / シャッター
- ファイアボルト / ファイアーボルト
- キュア・ウーンズ / キュアウーンズ

#### ルール2: 一般名詞の統一

❌ **NG:**
```markdown
第1章: 墓石
第2章: 墓碑
```

✅ **OK:**
```markdown
文脈に応じて統一（墓石 or 墓碑）
```

**注意:** 同じものを指す場合は統一。異なるものを指す場合は使い分けOK

### 3-3. 数字の表記統一

#### ルール1: 漢数字 vs アラビア数字

✅ **推奨:**
```markdown
- 1桁: 漢数字（一人、二本、三つ）
- 2桁以上: アラビア数字（10人、25本、100個）
- 慣用句: 漢数字（一瞬、二度と、三日月）
```

❌ **NG（ゆらぎ）:**
```markdown
第1章: 3人
第2章: 三人
```

✅ **OK:**
```markdown
すべて「三人」に統一（1桁は漢数字）
```

### 校正レビュー時のチェックポイント

**表現のゆらぎのチェックリスト:**
- [ ] 固有名詞（人名、地名、組織名）が統一されているか
- [ ] 呪文名が統一されているか
- [ ] 一般名詞が統一されているか
- [ ] 数字の表記が統一されているか

**校正レビュー例:**
```markdown
## 表現のゆらぎ

### ゆらぎ1
**箇所:** 「センサ船長」（第1章）と「センサー船長」（第3章）
**問題:** 同一人物の表記が不統一
**修正案:** 「センサ船長」に統一
```

## 4. 漢字・ひらがなバランス

### 4-1. 開く（ひらがなにする）べき語

以下の語は**ひらがな**で書くことを推奨します（読みやすさ重視）：

#### 接続詞・副詞

❌ **NG（漢字）:**
```markdown
更に、然し、又、尚、唯、僅か
```

✅ **OK（ひらがな）:**
```markdown
さらに、しかし、また、なお、ただ、わずか
```

#### 補助動詞

❌ **NG（漢字）:**
```markdown
見て見る、言って置く、やって行く
```

✅ **OK（ひらがな）:**
```markdown
見てみる、言っておく、やっていく
```

**ルール:** 補助動詞（「～ている」「～てみる」「～ておく」など）はひらがな

#### 形式名詞

❌ **NG（漢字）:**
```markdown
する事、見る物、行く時、その様な
```

✅ **OK（ひらがな）:**
```markdown
すること、見るもの、行くとき、そのような
```

**対象:** こと、もの、とき、ところ、ような、ため など

### 4-2. 閉じる（漢字にする）べき語

以下の語は**漢字**で書くことを推奨します（意味を明確にする）：

#### 動詞・名詞（意味が重要な語）

✅ **OK（漢字）:**
```markdown
見る、聞く、言う、思う、考える、感じる
戦い、剣、墓地、敵、仲間
```

❌ **NG（ひらがな）:**
```markdown
みる、きく、いう、おもう、かんがえる、かんじる
たたかい、つるぎ、ぼち、てき、なかま
```

**ルール:** 内容語（意味を持つ語）は漢字、機能語（文法的役割）はひらがな

### 4-3. バランスの取り方

#### ルール1: 連続する漢字は避ける

❌ **NG（漢字が多すぎる）:**
```markdown
巡礼者剣抜敵向
```

✅ **OK（ひらがなを挟む）:**
```markdown
巡礼者は剣を抜き、敵に向かった
```

**基準:** 漢字が4字以上連続する場合は、ひらがなを挟む

#### ルール2: ひらがなが続きすぎるのも避ける

❌ **NG（ひらがなが多すぎる）:**
```markdown
かれはつるぎをぬき、てきにむかった
```

✅ **OK（適切な漢字を使う）:**
```markdown
彼は剣を抜き、敵に向かった
```

**基準:** 漢字3-4割、ひらがな6-7割が読みやすい

### 校正レビュー時のチェックポイント

**漢字・ひらがなバランスのチェックリスト:**
- [ ] 接続詞・副詞がひらがなで書かれているか
- [ ] 補助動詞がひらがなで書かれているか
- [ ] 形式名詞がひらがなで書かれているか
- [ ] 漢字が4字以上連続していないか
- [ ] 全体的なバランスが適切か（漢字3-4割）

**校正レビュー例:**
```markdown
## 漢字・ひらがなバランス

### 問題1
**箇所:** 「更に彼は剣を抜いて見た」
**問題:** 「更に」「見た」が漢字で書かれている
**修正案:** 「さらに彼は剣を抜いてみた」
```

## 5. 誤字脱字

### 5-1. よくある誤字

#### パターン1: 入力ミス

❌ **NG:**
```markdown
彼女の瞳には確なかな覚悟が宿っていた
```

✅ **OK:**
```markdown
彼女の瞳には確かな覚悟が宿っていた
```

#### パターン2: 変換ミス

❌ **NG:**
```markdown
巡礼者は健を構えた
```

✅ **OK:**
```markdown
巡礼者は剣を構えた
```

**よくある変換ミス:**
- 健 → 剣
- 気が付く → 気がつく
- 声を掛ける → 声をかける

### 5-2. 脱字（文字の抜け）

❌ **NG:**
```markdown
巡礼者墓地に向かった
```

✅ **OK:**
```markdown
巡礼者は墓地に向かった
```

**よくある脱字:**
- 助詞（「は」「を」「に」）の抜け
- 句読点の抜け

### 5-3. 衍字（余計な文字）

❌ **NG:**
```markdown
巡礼者はは墓地に向かった
```

✅ **OK:**
```markdown
巡礼者は墓地に向かった
```

### 校正レビュー時のチェックポイント

**誤字脱字のチェックリスト:**
- [ ] 入力ミスがないか
- [ ] 変換ミスがないか
- [ ] 助詞の脱字がないか
- [ ] 句読点の脱字がないか
- [ ] 余計な文字（衍字）がないか

**校正レビュー例:**
```markdown
## 誤字脱字

### 誤字1
**箇所:** 「彼女の瞳には確なかな覚悟が宿っていた」
**問題:** 「確なかな」は「確かな」の誤字
**修正案:** 「彼女の瞳には確かな覚悟が宿っていた」
```

## 6. 同音異義語の誤用

### 6-1. よくある同音異義語の誤用

#### 1. 以外 / 意外

❌ **NG:**
```markdown
予想以外の展開
```

✅ **OK:**
```markdown
予想意外の展開（予想と違う）
巡礼者以外の全員（巡礼者を除く）
```

**使い分け:**
- **意外:** 予想と違う
- **以外:** ～を除く

#### 2. 作る / 造る / 創る

✅ **使い分け:**
- **作る:** 一般的な製作（料理を作る、計画を作る）
- **造る:** 大きなものを建造（船を造る、家を造る）
- **創る:** 新しいものを創造（世界を創る、芸術を創る）

#### 3. 会う / 合う / 遭う

✅ **使い分け:**
- **会う:** 人と会う
- **合う:** 合致する（意見が合う）
- **遭う:** 不運に遭遇（敵に遭う、災難に遭う）

#### 4. 聞く / 聴く / 訊く

✅ **使い分け:**
- **聞く:** 一般的（音を聞く、噂を聞く）
- **聴く:** 注意深く（音楽を聴く、講義を聴く）
- **訊く:** 質問する（理由を訊く）

**推奨:** リプレイ小説では基本的に「聞く」を使用（シンプル）

#### 5. 見る / 観る / 診る / 看る

✅ **使い分け:**
- **見る:** 一般的（景色を見る、様子を見る）
- **観る:** 鑑賞（映画を観る）
- **診る:** 診察（医者が診る）
- **看る:** 看病（病人を看る）

**推奨:** リプレイ小説では基本的に「見る」を使用

### 6-2. D&D用語の表記統一

#### 呪文名

✅ **統一表記:**
- 《シャター》（Shatter）
- 《ファイアボルト》（Fire Bolt）
- 《キュア・ウーンズ》（Cure Wounds）
- 《セイクリッド・フレイム》（Sacred Flame）

**ルール:** 《》で囲む、D&D公式準拠

#### 判定名

✅ **統一表記:**
- 攻撃ロール
- セーヴィング・スロー（セーブではない）
- 技能判定
- 能力値判定

### 校正レビュー時のチェックポイント

**同音異義語のチェックリスト:**
- [ ] 「以外/意外」が正しく使い分けられているか
- [ ] 「作る/造る/創る」が正しく使い分けられているか
- [ ] 「会う/合う/遭う」が正しく使い分けられているか
- [ ] 「聞く/聴く/訊く」が正しく使い分けられているか
- [ ] D&D用語が公式準拠で表記されているか

**校正レビュー例:**
```markdown
## 同音異義語の誤用

### 誤用1
**箇所:** 「予想以外の展開」
**問題:** 「以外」は「意外」の誤用
**修正案:** 「予想意外の展開」
**理由:** 「意外」= 予想と違う、「以外」= ～を除く
```

## 📊 校正レビューの評価基準

校正レビュー時には、誤り数で評価を行います。

| 評価 | 誤り数 | 対応 |
|---|---|---|
| **合格** | 0-5個（軽微なもののみ） | Phase 5へ |
| **要修正** | 6-15個 | 原作者が修正後Phase 5へ |
| **不合格** | 16個以上 | Phase 2へ戻って再執筆 |

### 誤りの重み付け

すべての誤りが同等ではありません。以下の重み付けで評価します：

- **文法エラー:** 2点（読解に影響）
- **句読点:** 1点
- **表現のゆらぎ:** 1点
- **漢字・ひらがなバランス:** 0.5点
- **誤字脱字:** 1点
- **同音異義語の誤用:** 2点（意味が変わる）

**合計が10点以上で「不合格」**

## 🎯 校正者としての心構え

1. **ルールベースで機械的に**
   - 感情は不要。ルールに従って淡々とチェック
   - 主観を排除し、客観的な誤りのみ指摘

   - 誤りのみを指摘し、文章の内容には踏み込まない

   - 細部まで丁寧に確認
