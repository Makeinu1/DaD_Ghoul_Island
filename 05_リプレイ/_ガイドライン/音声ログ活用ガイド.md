# 音声ログ活用ガイド

音声ログ（SRT形式）から特徴的な台詞や重要な会話を抽出するためのガイドです。

## SRTファイルの構造

SRTファイルは以下の形式になっています：

```
1
00:00:00,040 --> 00:00:03,040
字幕テキスト

2
00:00:03,840 --> 00:00:08,279
次の字幕テキスト
```

- 数字：字幕の番号
- 時間コード：開始時間 --> 終了時間
- テキスト：実際の台詞や会話

## 抽出すべき情報

### 1. キャラクターの特徴的な台詞

**例：**
- キャラクターの口癖や特徴的な言い回し
- 感情が強く表れている台詞
- キャラクターの性格が現れる発言

**検索方法：**
- キャラクター名で検索（「シャルル」「ノクシル」「巡礼者」など）
- 感情表現を含む文を探す（「嬉しい」「怒る」「悲しい」など）

### 2. NPCとの重要な会話

**例：**
- センサ船長との会話
- 重要な情報を伝える会話
- 関係性が深まる会話

**検索方法：**
- NPC名で検索（「センサ」「船長」「ルンジャタ」など）
- 重要なキーワードで検索（「失踪」「島」「異変」など）

### 3. プレイヤー間の印象的なやり取り

**例：**
- キャラクター同士の関係性が深まる会話
- 衝突や和解の場面
- 協力や信頼を示す会話

### 4. DMの描写で印象的な部分

**例：**
- 場面設定の描写
- 異形の登場シーン
- 緊張感のある場面の描写

## 抽出のコツ

### 雑談の見分け方

音声ログには以下のような雑談が多く含まれます：
- セッション開始前の準備の会話
- ルールの確認や説明
- 技術的な問題（配信、音声など）の会話
- ゲーム外の話題

**見分けるポイント：**
- 時間コードが早い（最初の数分）→ 準備の可能性が高い
- キャラクター名が出てこない → 雑談の可能性が高い
- 「配信」「音声」「ルール」などのキーワード → 雑談の可能性が高い

### 重要な会話の見分け方

以下の特徴がある場合は、重要な会話の可能性が高いです：
- キャラクター名が明確に出ている
- 感情表現が含まれている
- 物語の進行に関わる内容
- 過去の出来事や設定に言及している

## 実践例：第1回目の音声ログから

### 抽出した台詞例

**センサ船長とノクシルの再会：**
```
「ノクシル元気にしているかと」（センサ船長）
「船長がついに船長になったのか」（ノクシル）
「あなたが私のことを船長って呼んでくれてるのはも嬉しかったけどや、これで本当に私を船長になることはできたんだ」（センサ船長）
「楽しみにしてるよ。船長。」（ノクシル）
```

**活用方法：**
リプレイの「ノクシルとセンサ船長の再会」のシーンに、これらの台詞を組み込むことで、二人の関係性や感情がより鮮明になります。

## Cursorでの検索方法

### grep検索の例

```bash
# キャラクター名で検索
grep -i "シャルル\|ノクシル\|巡礼者" 第1回目-1.srt

# 特定のキーワードで検索
grep -i "船長\|センサ" 第1回目-1.srt

# 感情表現で検索
grep -i "嬉しい\|怒る\|悲しい\|驚く" 第1回目-1.srt
```

### コードベース検索の例

- 「第1回目のセンサ船長との会話」
- 「ノクシルとセンサの関係性を示す台詞」
- 「シャルルが怒っている場面」

## 注意事項

1. **録音失敗の可能性** - 全てのセッションに音声ログがあるわけではない
2. **雑談が多い** - 重要な部分のみを抽出する
3. **プレイヤーログとの整合性** - 矛盾がある場合はプレイヤーログを優先
4. **時間軸の確認** - SRTファイルの時間コードで、シーンの順序を確認できる

## リプレイへの組み込み方

抽出した台詞は、以下のようにリプレイに組み込みます：

```markdown
### ノクシルとセンサ船長の再会

**ノクシル：**
「船長がついに船長になったのか」

**センサ船長：**
「あなたが私のことを船長って呼んでくれてるのはも嬉しかったけどや、これで本当に私を船長になることはできたんだ」

**ノクシル：**
「楽しみにしてるよ。船長。」
```

台詞を直接引用することで、リプレイがより臨場感のあるものになります。
