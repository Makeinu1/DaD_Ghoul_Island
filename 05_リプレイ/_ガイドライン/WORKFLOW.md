# セッション後ワークフロー - 第12回以降

**作成日:** 2025-12-31
**目的:** 新しいセッション実施後、高品質なリプレイ小説を効率的に作成するための標準手順

---

## 🎯 ワークフロー全体像

```
セッション実施
    ↓
1. データ収集・整理
    ↓
2. リプレイ初稿作成
    ↓
3. 文体統一チェック
    ↓
4. セリフ洗練化
    ↓
5. キャラクター評価更新 ★NEW
    ↓
6. 最終レビュー
    ↓
完成・コミット
```

## 📥 ステップ1: データ収集・整理

### 1-1. プレイヤーログの配置

**実施内容:**
- 各PLから受け取ったログファイルを適切なフォルダに配置

**配置先:**
```
04_プレイヤーログ/
├── 巡礼者_PLチョモラン/第X回目.txt
├── ノクシル_PLたつや/第X回目.txt（あれば）
└── シャルル_PLまけいぬ/第X回目_シャルル日誌.txt（あれば）
```

**チェックリスト:**
- [ ] 巡礼者ログを受領・配置
- [ ] ノクシルログを受領・配置（あれば）
- [ ] シャルル日誌を受領・配置（あれば）
- [ ] ファイル名が統一形式か確認

### 1-2. 音声ログの配置

**実施内容:**
- セッション音声の文字起こしファイル（SRT形式）を配置

**配置先:**
```
07_セッション音声ログ/第X回目-1.srt
07_セッション音声ログ/第X回目-2.srt（複数ファイルの場合）
```

**チェックリスト:**
- [ ] 音声ログを配置
- [ ] 録音失敗がなかったか確認
- [ ] 複数ファイルの場合は連番で管理

### 1-3. DM公開情報の追加（あれば）

**実施内容:**
- DMから新しい世界観設定やNPC情報が公開された場合に配置

**配置先:**
```
02_DM公開情報/（新規ファイルまたは既存ファイルに追記）
```

## ✍️ ステップ2: リプレイ初稿作成

### 2-1. 一次データの確認

**優先順位:**
1. `04_プレイヤーログ/巡礼者_PLチョモラン/第X回目.txt`（最優先・時系列の主軸）
2. `04_プレイヤーログ/シャルル_PLまけいぬ/第X回目_シャルル日誌.txt`（内面描写）
3. `04_プレイヤーログ/ノクシル_PLたつや/第X回目.txt`（内面描写）
4. `07_セッション音声ログ/第X回目-*.srt`（特徴的な台詞のみ）

**チェックリスト:**
- [ ] 巡礼者ログで時系列を把握
- [ ] 他のPLログと照らし合わせて事実確認
- [ ] 音声ログから特徴的な台詞を抽出
- [ ] 矛盾点がないか確認

### 2-2. リプレイ初稿の執筆

**Claudeへの推奨プロンプト:**

```
第X回のリプレイ小説を執筆してください。

【必須条件】
- 一次データ（04_プレイヤーログ/）を最優先で参照
- 音声ログからは特徴的な台詞のみ抽出
- 複数のログで事実確認を行う
- リプレイテンプレート.mdの構成に従う

【参照すべきファイル】
1. 04_プレイヤーログ/巡礼者_PLチョモラン/第X回目.txt
2. 04_プレイヤーログ/シャルル_PLまけいぬ/第X回目_シャルル日誌.txt（あれば）
3. 04_プレイヤーログ/ノクシル_PLたつや/第X回目.txt（あれば）
4. 07_セッション音声ログ/第X回目-*.srt
5. 03_プレイヤー設定/シャルルストーリー/*（背景設定）
6. 05_リプレイ/_ガイドライン/リプレイテンプレート.md

【出力先】
05_リプレイ/第X回_（タイトル）.md
```

**チェックリスト:**
- [ ] リプレイテンプレートに従った構成
- [ ] 一次データに基づいた事実確認済み
- [ ] キャラクターの行動・台詞が正確
- [ ] 次回への伏線が明確

## 🎨 ステップ3: 文体統一チェック

### 3-1. 文体統一ガイドラインに基づくチェック

**参照ファイル:**
- `05_リプレイ/_ガイドライン/文体統一ガイドライン.md`

**チェック項目:**
- [ ] 地の文が「である調」で統一されているか
- [ ] 「ですます調」が混在していないか
- [ ] 主語が明確か（主語不明の文がないか）
- [ ] 短文・長文のバランスが取れているか
- [ ] 重要シーンは濃く、移動シーンは薄く描写されているか
- [ ] キャラクターの内面描写がプレイヤーログに基づいているか

### 3-2. Claudeへの修正依頼プロンプト

```
第X回のリプレイを、文体統一ガイドライン.mdに基づいて修正してください。

【参照ガイドライン】
05_リプレイ/_ガイドライン/文体統一ガイドライン.md

【確認項目】
- 地の文の「である調」統一
- 主語の明確化
- 短文・長文のバランス
- 描写の濃淡（重要シーン vs 移動シーン）
```

## 🗣️ ステップ4: セリフ洗練化

### 4-1. セリフ洗練化ガイドラインに基づくチェック

**参照ファイル:**
- `05_リプレイ/_ガイドライン/セリフ洗練化ガイドライン.md`

**チェック項目:**
- [ ] 自動文字起こしの誤変換を修正したか
- [ ] 不要なフィラー（「えっと」「その」「まあ」）を削除したか
- [ ] 口語的すぎる表現を修正したか
- [ ] 疑問符・感嘆符は適切に使用されているか
- [ ] キャラクターの口調・個性が反映されているか
- [ ] セリフの長さは適切か（長すぎず、短すぎず）

### 4-2. キャラクター別チェック

**巡礼者:**
- [ ] 古風で率直な物言いか
- [ ] 「矜持」「文化」などの言葉を適切に使用
- [ ] 現代口語（「マジで」等）を使っていないか

**ノクシル:**
- [ ] 寡黙でクールな口調か
- [ ] 「問題ない」などの口癖が適切に使用されているか
- [ ] 必要最小限の言葉で表現されているか

**シャルル:**
- [ ] パッション溢れる表現か
- [ ] 「義」「矜持」などの格調高い言葉を使用
- [ ] 正義感が表現されているか

### 4-3. Claudeへの修正依頼プロンプト

```
第X回のリプレイのセリフを、セリフ洗練化ガイドライン.mdに基づいて修正してください。

【参照ガイドライン】
05_リプレイ/_ガイドライン/セリフ洗練化ガイドライン.md

【確認項目】
- 音声ログの誤変換修正
- フィラー削除
- キャラクター別の口調チェック（巡礼者、ノクシル、シャルル）
- 疑問符・感嘆符の適切な使用
```

## 👥 ステップ5: キャラクター評価更新 ★NEW

### 5-1. PCキャラクター評価の更新

**目的:**
- 各セッションでのキャラクターの成長や新しい側面を記録
- 継続的なキャラクター理解の深化

**更新対象ファイル:**
```
08_LLMによるキャラ評/
├── PC_巡礼者.md
├── PC_ノクシル.md
└── PC_シャルル.md
```

**Claudeへの推奨プロンプト:**

```
第X回のセッションを踏まえて、以下のPCキャラクター評価を更新してください。

【対象キャラクター】
- 巡礼者
- ノクシル・グリムホルト
- シャルル・ビアトリス

【参照ファイル】
1. 04_プレイヤーログ/（各PLのログ）
2. 05_リプレイ/第X回_*.md（完成版リプレイ）
3. 08_LLMによるキャラ評/PC_*.md（既存評価）

【更新内容】
- 第X回での新しい側面や成長
- 他PCやNPCとの関係性の変化
- 重要な決断や行動
- 内面的な変化（シャルル日誌、ノクシルログがあれば）

【更新形式】
各ファイルの末尾に「## 第X回での変化」セクションを追加
既存の総評も必要に応じて更新
```

**チェックリスト:**
- [ ] 巡礼者の評価を更新
- [ ] ノクシルの評価を更新
- [ ] シャルルの評価を更新
- [ ] 一次データに基づいた内容か確認

### 5-2. NPCキャラクター評価の更新・追加

**更新対象ファイル:**
```
08_LLMによるキャラ評/
├── NPC_センサ.md
├── NPC_ネギ.md
├── NPC_ディードリド.md
├── NPC_セラフィナ.md
├── NPC_ルンジャタ.md
└── NPC_シマ.md
（新規NPCが登場した場合は新規ファイル作成）
```

**Claudeへの推奨プロンプト:**

```
第X回のセッションに登場したNPCについて、キャラクター評価を更新または新規作成してください。

【登場NPC】
- （第X回に登場したNPC名をリスト）

【参照ファイル】
1. 04_プレイヤーログ/（各PLのログ）
2. 05_リプレイ/第X回_*.md（完成版リプレイ）
3. 08_LLMによるキャラ評/NPC_*.md（既存評価）

【更新・作成内容】
- 既存NPCの場合: 第X回での新しい情報、関係性の変化
- 新規NPCの場合: 基本情報、特徴、PCとの関係性、役割

【更新形式】
既存NPC: ファイル末尾に「## 第X回での動向」セクションを追加
新規NPC: NPC_（名前）.md を新規作成
```

**チェックリスト:**
- [ ] 第X回に登場した既存NPCの評価を更新
- [ ] 新規登場NPCの評価を作成
- [ ] PCとの関係性が正確に記録されているか
- [ ] 一次データに基づいた内容か確認

## ✅ ステップ6: 最終レビュー

### 6-1. 全体の整合性チェック

**チェック項目:**
- [ ] 時系列の矛盾がないか
- [ ] キャラクター名・地名の表記が統一されているか
- [ ] 前回からの伏線が適切に処理されているか
- [ ] 次回への伏線が明確に示されているか
- [ ] 文体統一ガイドラインに準拠しているか
- [ ] セリフ洗練化ガイドラインに準拠しているか

### 6-2. 一次データとの最終照合

**チェック項目:**
- [ ] プレイヤーログと矛盾する記述がないか
- [ ] 重要な判定結果が正確に記録されているか
- [ ] キャラクターの行動・発言が正確か
- [ ] 創作部分（プレイヤーログにない内容）が含まれていないか

### 6-3. 品質チェックリスト

**文体:**
- [ ] である調で統一されている
- [ ] 主語が明確
- [ ] 短文・長文のバランスが良い

**セリフ:**
- [ ] キャラクターの個性が表現されている
- [ ] 不要なフィラーが削除されている
- [ ] 音声ログの誤変換が修正されている

**内容:**
- [ ] 一次データに基づいている
- [ ] 時系列が正確
- [ ] 伏線が適切に処理されている

**キャラクター評価:**
- [ ] PCの評価が最新情報に更新されている
- [ ] NPCの評価が更新されている
- [ ] 新規NPCの評価が作成されている
