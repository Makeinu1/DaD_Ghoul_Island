# セッション後ワークフロー

詳細は @05_リプレイ/_ガイドライン/WORKFLOW.md を参照。

---

## クイックワークフロー

```
1. 一次データ収集
   └─ プレイヤーログ、音声ログを確認

2. リプレイ初稿作成
   ├─ 一次データを最優先で参照
   └─ 既存キャラ評は「参考」として使用（鵜呑みにしない）

3. ユーザーレビュー・修正
   └─ 事実誤認、話者エラーの修正

4. キャラ評更新（リプレイ完成後）
   ├─ 新しい情報をキャラ評に反映
   └─ 既存の誤りがあれば修正

5. Webサイト・Git反映
   ├─ リプレイをsessionXX.md形式でコピー
   ├─ mkdocs.yml、index.md更新
   ├─ ビルド・デプロイ
   ├─ **デプロイ後にURLアクセスして表示確認** ← 重要
   └─ コミット・プッシュ
```

## 協働型マルチエージェントワークフロー

第12回以降は以下のPhaseで進行：

| Phase | 担当 | 内容 |
|---|---|---|
| 0 | ユーザー | 素材準備（時系列メモ、セリフ抽出） |
| 1 | 原作者↔編集者 | 構成設計 |
| 2 | 原作者 | 初稿執筆 |
| 3 | 編集者 | 読み物クオリティチェック |
| 4 | 校正者 | 文章正確性チェック |
| 5 | 検証者 | 一次データとの事実照合 |
| 6 | ユーザー | 最終確認 |
| 7 | キャラ評エージェント | キャラクター評価更新 |
| 8 | - | 完成・公開（Git・Webサイト） |

---

## 重要: キャラ評更新のタイミング

- ❌ 旧: キャラ評を「最初に」更新 → 既存の誤りを引き継ぐリスク
- ✅ 新: 一次データを読む → リプレイ作成 → キャラ評更新の順序
